# Headless UI and Design Systems

## Definicao curta

Separar contrato e comportamento da camada visual, permitindo que o design
system mude sem redefinir a semantica do runtime.

## Onde aparece no Praxis

- Frontend: `@praxisui/core` para modelos e metadados
- Frontend: `@praxisui/dynamic-fields` para implementacoes concretas
- Frontend: `@praxisui/settings-panel` para editores configuraveis

## Referencias publicas

- repositório público: `https://github.com/codexrodrigues/praxis-ui-angular`
- pacotes: `@praxisui/core`, `@praxisui/dynamic-fields`, `@praxisui/settings-panel`
