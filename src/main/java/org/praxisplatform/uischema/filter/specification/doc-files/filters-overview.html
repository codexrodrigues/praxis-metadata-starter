Visão Geral de Filtros

Este documento explica como o mecanismo de filtros converte DTOs anotados com @Filterable em Specifications JPA, cobrindo as operações suportadas, boas práticas e considerações de uso corporativo.

Arquitetura

  • DTOs de Filtro: implementam GenericFilterDTO e declaram campos com @Filterable.
  • GenericSpecificationsBuilder: lê os DTOs e monta predicados (Criteria API) conforme as operações.
  • Services: consomem a Specification resultante para executar consultas paginadas/ordenadas.

Operações Suportadas (principais)

  • Igualdade e comparação: EQUAL, NOT_EQUAL, GREATER_THAN, GREATER_OR_EQUAL, LESS_THAN, LESS_OR_EQUAL
  • Strings (case-insensitive): LIKE, NOT_LIKE, STARTS_WITH, ENDS_WITH
  • Listas: IN, NOT_IN
  • Intervalos: BETWEEN (inclusivo), BETWEEN_EXCLUSIVE (exclusivo), NOT_BETWEEN, OUTSIDE_RANGE
  • Datas/relativos: ON_DATE (usa janela do dia), IN_LAST_DAYS, IN_NEXT_DAYS
  • Nulos/booleanos: IS_NULL, IS_NOT_NULL, IS_TRUE, IS_FALSE
  • Coleções: SIZE_EQ, SIZE_GT, SIZE_LT (requer relation para atributo de coleção)

Notas de Modelagem

  • IS_NULL/IS_NOT_NULL: modele como Boolean e envie true para ativar o predicado.
  • IS_TRUE/IS_FALSE: o predicado é aplicado quando o campo do DTO é fornecido (recomenda-se enviar true).
  • SIZE_* (coleções): aponte relation para um atributo de coleção (ex.: "itens"); caso contrário, o builder lança erro informativo.
  • Datas: ON_DATE/IN_LAST_DAYS/IN_NEXT_DAYS normalizam para Instant e horário atual; padronize em UTC em ambientes multi-região.

Exemplo de DTO de Filtro (trecho)

public class PedidoFilterDTO implements GenericFilterDTO {
  @Filterable(operation = FilterOperation.LIKE)
  private String clienteNome;

  @Filterable(operation = FilterOperation.GREATER_OR_EQUAL)
  private BigDecimal valorMin;

  @Filterable(operation = FilterOperation.ON_DATE)
  private LocalDate criadoEm;

  @Filterable(operation = FilterOperation.SIZE_GT, relation = "itens")
  private Integer quantItensMaiorQue;

  @Filterable(operation = FilterOperation.IS_TRUE)
  private Boolean ativo;
}

Ordenação (Sort) com Relações

Para ordenar por um atributo relacionado, use o nome do campo do DTO no parâmetro sort da requisição. O builder usa o relation da anotação @Filterable para mapear corretamente. 

POST /api/pedidos/filter?page=0&size=20&sort=clienteNome,asc
{
  "clienteNome": "ana"
}

Boas Práticas Corporativas

  • Prefira filtros seletivos (evitar IN/NOT_IN com listas muito grandes).
  • Garanta índices para colunas frequentemente filtradas ou ordenadas.
  • Padronize timezone (UTC) para consistência entre backends/banco.
  • Documente relações usadas em relation e padrões de ordenação expostos.
